# API Documentation

## API Enpoints

### /inference
*Performs inference of a pre-trained model.*

http://localhost:8080/inference

- **user_code:** string containing the user's unique code; if defined, it will read the user's models and public models; if not defined, it will only read public models (default value: '')
- **model_name:** string with the name to the ".model" or ".sapiens" model that will be loaded (default value: '')
- **messages:** array of objects with the messages from the dialogue with the model (default value: [])
	- **role:** string that accepts "system" for system instructions, "user" for user messages, or "assistant" for model responses
	- **content:** string containing the message content
	- **file_paths:** array of strings containing the local or web paths to files that need to be interpreted (optional)
	- **base64:** array of files objects in base64 (alternative to "file_paths")
		- **base64_string:** string with the file code in base64
		- **type:** string with the file extension without the dot
- **max_tokens:** integer with the approximate number for the maximum limit of tokens returned (default value: None)
- **stream:** if true, it will return a tokens generator so that the response can be built in real time; if false, it will return a dictionary with the inference completion (default value: true)
- **language:** string with the code of the preferred language for the model's responses (default value: '')
- **temperature:** floating number with a percentage scale between 0 and 1 referring to the level of randomness between one inference and another (default value: 0.5)
- **creativity:** floating number with a percentage scale between 0 and 1 referring to the level of creativity in the model's inferences (default value: 0.5)
- **humor:** floating number with a percentage scale between 0 and 1 referring to the mood level in the model's inferences (default value: 0.5)
- **formality:** floating number with a percentage scale between 0 and 1 referring to the level of formality in the model's inferences (default value: 0.5)
- **political_spectrum:** floating number with a percentage scale between 0 and 1 referring to the political spectrum contained in the model's responses; the closer to 0, the greater the left bias; the closer to 1, the greater the right bias (default value: 0.5)
- **reasoning:** floating number with a percentage scale between 0 and 1 referring to how much reasoning the model will apply before returning the answer's conclusion (default value: 0.0)
- **reflection:** if true, the model will reflect on your final answer and may improve it if necessary; if false, the model will not reflect on your answer (default value: false)
- **tools:** array of objects with zero, one, or more functions that the model may call to execute external tools (default value: [])
- **progress:** if true, it will display the loading progress; if false, it will not display the progress (default value: true)

**Pattern of return json:**

- **success:** returns true if the execution is successful, or false otherwise
- **inference:** returns a JSON object with the inference result containing the following keys:
	- **answer:** string with the response from the model (complete response if "stream" is false, or response under construction if "stream" is true)
	- **files:** returns a array of files generated in the following json format: {'base64_string': 'string with the file code in base64...', 'type': 'string with the file extension...'}
	- **sources:** returns a array of dictionaries with sources consulted by the model on the web; the dictionaries will have the following format: {'domain': 'string with the domain address...', 'url': 'string with the full url address...'}
	- **javascript_charts:** array of strings with html codes of charts generated by the model
	- **artifacts:** array of strings with html codes of artifacts generated by the model
	- **next_token:** string with the last token generated by the model (the current token)
	- **tool_call:** json array with the results of the function calls in the following format: {'function_call': 'string with the name of the function to be executed...', 'arguments': [{'name': 'parameter_name_N': 'value of parameter N'}]}
	
	*The keys "answer_for_files" and "str_cost" are for internal control and can be ignored.*

**Pattern of tools send:**

- **function_name:** string with the name of the function that will be called
- **description:** string with the description of the function, its usefulness, and its return type
- **parameters:** array of objects containing the structure of the function parameters
	- **name:** string containing the name of the function parameter
	- **type:** string containing the parameter type name
	- **default:** default value of any type, containing the value that should be used if this parameter is not received
	- **description:** string describing the parameter, its purpose, and when it should be extracted from the prompt


#### cURL
In this case, the server will look in the current directory for a default folder named "models" and search inside this folder for a model named "my_model_name.sapiens". If the "models" folder is not in the current execution directory of the script, then you will need to specify the path to the directory in the "models_path" parameter of the "runAPI" function. The path in "models_path" does not include the name of the "models" folder, it only specifies the path to reach the directory that contains it.

In the following example, there must be the path "./models/my_model_name.sapiens" (in the directory where the script for executing the function "runAPI" is located).
```cURL
curl -sN -X POST "http://localhost:8080/inference" \
-H "Content-Type: application/json" \
-d '{
	"user_code": "",
	"model_name": "my_model_name.sapiens",
	"messages": [{"role": "user", "content": "Hello! How are you?"}],
	"max_tokens": null,
	"stream": true,
	"language": "",
	"temperature": 0.5,
	"creativity": 0.5,
	"humor": 0.5,
	"formality": 0.5,
	"political_spectrum": 0.5,
	"reasoning": 0.0,
	"reflection": false,
	"tools": [],
	"progress": true
}' | sed 's/}{/}\n{/g' | jq -r '.next_token // empty' | tr -d '\n'; echo
```

```html
<!DOCTYPE html>
<html lang="en">
<body>
    <script>
        (async () => {
            const response = await fetch('http://localhost:8080/inference', {
                method: 'POST',
                headers: {
                  'Content-Type': 'application/json'
                },
                body: JSON.stringify({
					user_code: '',
                    model_name: 'my_model_name.sapiens',
                    messages: [
						{ role: 'user', content: 'Hello! How are you?' }
                    ],
                    max_tokens: null,
                    stream: true,
                    language: '',
                    temperature: 0.5,
                    creativity: 0.5,
                    humor: 0.5,
                    formality: 0.5,
                    political_spectrum: 0.5,
                    reasoning: 0.0,
                    reflection: false,
                    tools: [],
					progress: true
                })
            });

            const reader = response.body.getReader();
            const decoder = new TextDecoder();
            let buffer = '';
            let output = '';

            while (true) {
                const { done, value } = await reader.read();
                if (done) break;

                buffer += decoder.decode(value, { stream: true });
                const parts = buffer.split('\n');
                buffer = parts.pop();

                for (let part of parts) {
                    if (!part.trim()) continue;
                    try {
                        const json = JSON.parse(part);
                        if (json.next_token) {
                            output += json.next_token;
                            document.body.innerText = output;
                        }
                    } catch (e) {}
                }
            }
        })();
    </script>
</body>
</html>

```

```html
    <script>
        (async () => {
            async function getCryptoPrice(coinName='', currencyCode='') {
                const url = `https://api.coingecko.com/api/v3/simple/price?ids=${coinName}&vs_currencies=${currencyCode}`;
                const response = await fetch(url);
                const data = await response.json();
                return data;
            }

            const coinNameDefault = 'bitcoin';
            const currencyCodeDefault = 'usd';

            const response = await fetch('http://localhost:8080/inference', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify({
                    user_code: '',
                    model_name: 'function-model-7b-1m.sapiens',
                    messages: [
                        { role: 'user', content: 'What is the current value of Ethereum in Brazilian reais?' }
                    ],
                    stream: true,
                    tools: [
                        {
                            function_name: 'get_crypto_price',
                            description: 'Returns the current price of a cryptocurrency.',
                            parameters: [
                                { name: 'coin_name', type: 'string', default: coinNameDefault, description: 'Receives a string with the name of the cryptocurrency to be quoted.' },
                                { name: 'currency_code', type: 'string', default: currencyCodeDefault, description: 'Receives a string containing the exchange rate currency code.' }
                            ]
                        }
                    ]
                })
            });

            const data = await response.json();
            const toolCall = data.tool_call || [];

            for (const func of toolCall) {
                const functionCall = func.function_call || '';
                if (functionCall === 'get_crypto_price') {
                    const argumentsList = func.arguments || [];

                    let coinName = '';
                    let currencyCode = '';

                    for (const argument of argumentsList) {
                        if (!coinName && argument.coin_name) coinName = argument.coin_name || coinNameDefault;
                        else if (!currencyCode && argument.currency_code) currencyCode = argument.currency_code || currencyCodeDefault;
                    }

                    const cryptoData = await getCryptoPrice(coinName, currencyCode);
                    const cryptoPrice = cryptoData[coinName][currencyCode];

                    document.body.innerText += `The current price of ${coinName} in ${currencyCode} is: ${cryptoPrice}\n`;
                }
            }
        })();
    </script>
</body>
```

### /training
*Performs the training of a base model.*

http://localhost:8080/training

- **user_code:** string containing a unique code to identify the user; if defined, it will create a folder using this code as the name and will use this folder to read and save the user's models; if not defined, it will use the default folder named "models" to read and save the models (default value: '')
- **base64_dictionary:** json in the format: {'base64_string': 'string with the file code in base64...', 'type': 'file extension without the dot...'}; replace "dataset_path" (default value: {})
- **dataset_path:** string with the path to a file with training data, with a directory path containing multiple files, with a path to a zip or tar.gz package, with the address of a web page, or with the url of a youtube video (default value: '')
- **presets:** if true, it will use the internet connection to download pre-configured parameters so that the model starts training with basic language and conversation skills; if false, it will not download the presets (default value: false)
- **supplements:** if true, it will use the internet connection to download pre-configured parameters with additional supplements so that the model starts training with improved language and conversation skills; if false, it will not download the supplements (default value: false)
- **model_name:** string containing the name of the file that will be generated with the trained model; the file will be generated in the "models" folder if the user code is not specified; if the user code is specified, the model will be generated within a folder whose name is the user code itself (default value: '')
- **progress:** if true, it will display the training progress; if false, it will not display the progress (default value: true)

**Pattern of return json:**

- **success:** returns true if the execution is successful, or false otherwise

#### cURL
In this case, a model named "base_model.model" will be generated in the "models" folder.

In the example, there must be a file "dataset01.txt" in a "datasets" folder in the script's execution directory.
The completion of the process can take from seconds to minutes, or even hours if the dataset is very large.
```cURL
curl -sN -X POST "http://localhost:8080/training" \
-H "Content-Type: application/json" \
-d '{
	"user_code": "",
	"base64_dictionary": {},
	"dataset_path": "./datasets/dataset01.txt",
	"presets": false,
	"supplements": false,
	"model_name": "base_model",
	"progress": true
}'; echo
```

```bash
{"success": true}
```

### /continuous-pre-training

http://localhost:8080/continuous-pre-training

- **output_name:** name of the model that will be generated with the result of continued pre-training; the model will be generated in "models" if the user code is not defined, or in the user's folder if the user code is defined (default value: '')
- **progress:** if true, it will display the training progress; if false, it will not display the progress (default value: true)

### /reload-model
*Reloads a user's private model.*

http://localhost:8080/reload-model

- **user_code:** string containing the user code whose model will be reloaded (default value: '')
- **model_name:** string containing the name of the model from the "models" folder that should be reloaded (default value: '')

**Pattern of return json:**

- **success:** returns true if the execution is successful, or false otherwise

#### cURL
This endpoint differs from the previous one by making it possible to define the user code when there are models within "specific users" folders that need to be reloaded. While "/reload-sapiens-model" reloads public ".sapiens" models for all users, "/reload-model" only has permission to reload models belonging to the current user.
```cURL
curl -sN -X POST "http://localhost:8080/reload-model" \
-H "Content-Type: application/json" \
-d '{
	"user_code": "c1e728bc75721a24c2c50d8d9822657359f0cfa34c89500dd19defb3d434ad81",
	"model_name": "base_model.model"
}'; echo
```
In this example, the return value will be "false" because we did not create any models folders for the specified user.
```bash
{"success": false}
```
Now we will move the model file into a folder with the specified user code, so that it can be reloaded.
```python
from os import makedirs
from os.path import exists, join
from shutil import move
source_path = join('models', 'base_model.model')
destination_dir = 'c1e728bc75721a24c2c50d8d9822657359f0cfa34c89500dd19defb3d434ad81'
```

### /download-model
*Downloads a user model.*

http://localhost:8080/download-model

- **user_code:** string containing the user code whose model will be downloaded (default value: '')
- **model_name:** string containing the name of the model that should be downloaded (default value: '')

**Pattern of return json:**

- **success:** returns true if the execution is successful, or false otherwise

#### cURL
In this case, the "base_model.model" model located in the user's folder "c1e728bc75721a24c2c50d8d9822657359f0cfa34c89500dd19defb3d434ad81" will be downloaded. Only user models can be downloaded.

First, we'll create a sample model for a user.
```cURL
curl -sN -X POST "http://localhost:8080/training" \
-H "Content-Type: application/json" \
-d '{
	"user_code": "c1e728bc75721a24c2c50d8d9822657359f0cfa34c89500dd19defb3d434ad81",
	"base64_dictionary": {},
	"dataset_path": "./datasets/dataset01.txt",
	"presets": false,
	"supplements": false,
	"model_name": "base_model",
	"progress": true
}'; echo
```
```bash
{"success": true}
```
Now we can download the model created for the user in question.
#### JavaScript
```html
<!DOCTYPE html>
<html lang="en">
<body>
    <script>
    (async () => {
        model_name = 'base_model.model'
        const response = await fetch('http://localhost:8080/download-model', {
            method: 'POST',
            headers: {'Content-Type': 'application/json'},
            body: JSON.stringify({
                user_code: 'c1e728bc75721a24c2c50d8d9822657359f0cfa34c89500dd19defb3d434ad81',
                model_name: model_name
            })
        });

        if (!response.ok) throw new Error('Download error.');

        const blob = await response.blob();
        const url = URL.createObjectURL(blob);

        const link = document.createElement('a');
        link.href = url;
        link.download = model_name;
        link.click();

        URL.revokeObjectURL(url);
    })();
    </script>
</body>
</html>

```
